## 1. Product Overview

**MyTournament.PB** is Malaysia's independent pickleball tournament calendar — a faceless media brand that aggregates every verified tournament from all platforms into one place, with deadline tracking, skill-level filtering, and weekly content drops. The brand operates as **Admin PB**, the invisible group-chat admin who checks every platform so players don't have to.

- **Live site:** https://nainaii1.github.io/mytournament-pb/ (GitHub Pages; production domain mytournamentpb.my pending)
- **Instagram:** @mytournament.pb · **Facebook:** @mytournamentpb · **Threads:** @mytournament.pb
- **Data source:** Google Sheet (gviz CSV endpoint, live on page refresh)
- **Founded:** May 2026, Kuala Lumpur

---

## 2. Problem Statement

Malaysian pickleball players must manually check four separate platforms — Sportssync, Baseline, Sports We Play, and Reclub — to find all upcoming tournaments. There is no single trusted source. Registration deadlines close silently, skill-level brackets are inconsistently labelled across platforms, and there is no community layer for finding doubles partners or evaluating value (entry fee vs. prize pool). The result: players miss tournaments they qualify for, register for the wrong category, or discover a deadline has passed.

As the Malaysian pickleball community grows rapidly (KL positioned as SEA tournament hub in 2026), information fragmentation is the #1 friction point between players and participation.

## 4. Goals

1. **Zero missed tournaments** — A player who checks MyTournament.PB weekly never misses a tournament they qualify for
2. **Zero missed deadlines** — Closing Soon alerts ensure players always have 72+ hours to decide

4. **Value clarity** — Know immediately whether a tournament is worth the entry fee

## 6. User Stories

### Player — Finding Tournaments

- As a **competitive player**, I want to see all upcoming tournaments sorted by date so that I can plan my season without checking four separate platforms.
- As a **novice player**, I want to filter tournaments by skill level so that I only see events I'm eligible to enter.
- As a **player planning June**, I want to filter by month so that I can see only June tournaments without scrolling past May.
- As a **mobile user**, I want tournament cards to load and filter fast on my phone so that I can check during a match break or group chat conversation.

### Player — Deadlines

- As a **busy player**, I want to see a "closing soon" alert so that I don't miss the last 72 hours before registration closes.
- As a **registered player**, I want to immediately see whether registration is open or closed for any tournament so that I don't waste time clicking dead links.

### Player — Value Assessment

- As a **budget-aware player**, I want to see entry fee AND prize pool on the same card so that I can decide whether a tournament is worth it before clicking through.
- As a **beginner**, I want an explainer of who can join a big tournament so that I know if it's right for my level.

### Player — Community

- As a **doubles player** without a partner, I want a community board where I can signal I'm looking for a partner so that I can find someone before registration closes.

### Organizer — Getting Listed

- As an **independent organizer**, I want to submit my tournament for listing so that players from all platforms can find it.
- As a **platform organizer** (Sportssync/Baseline/SWP), I want my tournaments automatically picked up from my platform so that I get free incremental reach.

### Admin (Internal — Founder Workflow)

- As **Admin PB**, I want the Google Sheet to be the single source of truth so that updating one place updates the website instantly.
- As **Admin PB**, I want a weekly deadline audit so that I never miss flagging a closing-soon event.
- As **Admin PB**, I want a weekly scan of all platforms so that no tournament goes unlisted.

## 7. Product Requirements

### Website — Phase 0 (CURRENT — target May 31)

**P0 — Must Have (cannot ship without)**

| Requirement | Acceptance Criteria |
|-------------|---------------------|
| Tournament list sorted by start date | All Verified/Published rows shown ascending by Start Date |
| Tournament card: name, dates, venue, state, entry fee, prize pool, skill level, platform pill, register link | All fields visible on 390px screen without horizontal scroll |
| Closing Soon amber strip | Auto-shows when any tournament has deadline ≤7 days; links to that section |
| Open / Closed registration state | Dimmed cards for past deadline; clear badge on open tournaments |
| THE PICK / FEATURED badge | Rows with Pick Priority "1 — THE PICK" show badge |
| Month filter tabs (ALL/MAY/JUN/JUL) | Dynamic from data; tapping a month filters all sections |
| Skill level filter chips (multi-select) | Novice/Intermediate/Advanced/Open; multiple can be active simultaneously |
| Mobile-first layout | No horizontal scroll at 390px; touch targets ≥44px |
| Platform-labelled register button | "Register · Sportssync →", "Register · Baseline →" etc. |
| Footer with social links | Instagram, Facebook, TikTok, email |

### Data Layer

**P0 — Core data requirements**

| Requirement | Detail |
|-------------|--------|
| Google Sheet as single source of truth | Status = Verified OR Published shows on site |
| 25-column schema | Status, ID, Tournament Name, Organizer, Sponsor, Start/End Date, Reg Deadline, State, Venue, Entry Fee (RM), Prize Pool (RM), Cash Prize (RM), Merch (RM), Prize Note, Skill Level, Event Type, Pick Priority, Editorial Angle, Format Note, Player Note, Source Platform, Reg URL, Date Added, Last Checked, Notes |
| Date normalisation | All dates YYYY-MM-DD; D-Mon-YYYY auto-normalised |
| DUPR skill level parsing | Novice (<3.0), Intermediate (<3.5), Advanced (<4.0), Advanced+ (<4.5), Open (no cap) |
| isClosed / isClosingSoon computed fields | Deadline past OR "Closed"/"Once full" = closed; deadline ≤7 days = closing soon |
| Monday + Thursday data sweeps | Manual; Sportssync + Baseline + SWP + Reclub checked each sweep |
